# Comprehensive Source Discovery Checklist — AI / Robot Layoffs Data Hunt

Use this as a **maximum-depth one-time source discovery brief** for the project.

The objective is to find **every meaningful source class, search angle, dataset, archive, article type, document type, public filing, and monitoring path** that could contribute to the trackers, so the system does not stop after finding only a handful of obvious examples.

This is a **discovery exhaustion task**.

The goal is not speed. The goal is **coverage**.

---

# 1. Core Objective

Perform a deep, methodical, highly redundant search for all possible sources relevant to:

- AI-attributed layoffs
- AI-linked job cuts
- job losses where AI is explicitly cited
- job losses where AI is likely a contributing factor
- robot / automation / humanoid / warehouse automation job displacement
- AI-driven workforce restructuring
- AI-driven hiring freezes if materially relevant
- AI-created jobs
- productivity gains attributed to AI
- labor reduction claims tied to AI efficiency
- company statements about replacing, reducing, or restructuring labor using AI or automation

The output should become the **master source map** for the project.

---

# 2. Discovery Mindset

Do **not** stop when you find enough data to populate the site.

Do **not** assume the most obvious sources are sufficient.

Do **not** terminate search because a few strong examples were found.

Treat this as a **coverage audit**.

Assume valuable sources may exist in:
- obscure regions
- trade publications
- filings
- earnings call transcripts
- press release archives
- niche industry journals
- local-language reporting
- government datasets
- labor organizations
- consultancy reports
- academic datasets
- workforce studies
- company newsroom pages
- investor presentations
- court documents
- union statements
- job posting trend reports
- automation vendor case studies

Search until source categories are exhausted, not just until examples are found.

# 4. Search Completion Standard

This task is not complete until the search has covered **all major source classes** below and explicitly reported:

- what was found
- what was not found
- what appears missing globally
- what is strong enough for ongoing ingestion
- what is only useful as validation

If a category produces weak results, record that fact and move on.
Do not silently skip categories.

# 10. Source Validation Checklist

For every promising source, assess:

- Is it original or derivative?
- Does it contain numeric job counts?
- Does it cite AI explicitly or only imply it?
- Does it contain enough evidence for classification?
- Is it machine-readable?
- Is it stable enough for repeated ingestion?
- Does it have RSS, sitemap, archive, or search?
- Is it paywalled?
- Is it legally safe to summarize?
- Is it likely to disappear?
- Does it need manual review?
- Can it help with dedupe?
- Does it tend to inflate, understate, or sensationalize?

# 16. Completion Rule

Only consider the search complete when:
- every major source class above has been explicitly checked
- weak categories are reported, not skipped
- source quality has been graded
- regional gaps are documented
